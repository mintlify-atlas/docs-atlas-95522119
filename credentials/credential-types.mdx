---
title: "Credential Types"
description: "Reference for every AWSCredentials subclass in the AWS SDK for .NET, with usage examples."
---

All credential objects in the AWS SDK for .NET inherit from the abstract `AWSCredentials` class, which exposes a single `GetCredentials()` method that returns an `ImmutableCredentials` snapshot (access key, secret key, optional session token). The SDK calls this method automatically before each request, so refreshing credential types like `AssumeRoleAWSCredentials` stay current without any action on your part.

## Credential type reference

<AccordionGroup>
  <Accordion title="BasicAWSCredentials">
    Use `BasicAWSCredentials` when you have a static access key and secret key — for example, an IAM user's long-term credentials stored in a secrets manager or injected at startup.

    The optional third constructor parameter accepts a 12-digit AWS account ID. When set, and the target service supports account-based endpoints, the SDK routes requests through the account-specific endpoint (`https://<accountId>.ddb.region.amazonaws.com`) instead of the regional endpoint.

    ```csharp
    using Amazon.Runtime;
    using Amazon.S3;

    // Access key + secret key only
    var credentials = new BasicAWSCredentials(
        accessKey: "AKIAIOSFODNN7EXAMPLE",
        secretKey: "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
    );

    var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USEast1);
    ```

    ```csharp
    // With optional account ID for account-based endpoint routing
    var credentials = new BasicAWSCredentials(
        accessKey: "AKIAIOSFODNN7EXAMPLE",
        secretKey: "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY",
        accountId: "123456789012"
    );
    ```

    <Warning>
      `BasicAWSCredentials` stores keys in plain memory and never refreshes them. Do not use it with short-lived credentials; use `SessionAWSCredentials` for those.
    </Warning>
  </Accordion>

  <Accordion title="SessionAWSCredentials">
    Use `SessionAWSCredentials` when you have temporary credentials that include a session token. These are typically obtained from STS `AssumeRole`, `GetSessionToken`, or the EC2 instance metadata service.

    ```csharp
    using Amazon.Runtime;
    using Amazon.S3;

    var credentials = new SessionAWSCredentials(
        awsAccessKeyId: "ASIAIOSFODNN7EXAMPLE",
        awsSecretAccessKey: "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY",
        token: "AQoDYXdzEJr//////////wEa..."
    );

    var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USEast1);
    ```

    `SessionAWSCredentials` is a concrete, non-refreshing credential type. If the token expires, subsequent requests will fail with an authentication error. For credentials that automatically refresh before expiry, use `AssumeRoleAWSCredentials` instead.
  </Accordion>

  <Accordion title="EnvironmentVariablesAWSCredentials">
    `EnvironmentVariablesAWSCredentials` reads credentials directly from process environment variables every time `GetCredentials()` is called:

    | Variable | Purpose |
    |---|---|
    | `AWS_ACCESS_KEY_ID` | Access key ID |
    | `AWS_SECRET_ACCESS_KEY` | Secret access key |
    | `AWS_SESSION_TOKEN` | Session token (optional) |
    | `AWS_ACCOUNT_ID` | Account ID (optional) |

    The constructor throws `InvalidOperationException` immediately if `AWS_ACCESS_KEY_ID` or `AWS_SECRET_ACCESS_KEY` are not set, making misconfiguration obvious at startup rather than at the first API call.

    ```bash
    export AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
    export AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
    export AWS_SESSION_TOKEN=AQoDYXdzEJr...   # optional
    ```

    ```csharp
    using Amazon.Runtime;
    using Amazon.S3;

    // Reads AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_SESSION_TOKEN from the environment
    var credentials = new EnvironmentVariablesAWSCredentials();
    var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USEast1);
    ```

    The credential provider chain uses `EnvironmentVariablesAWSCredentials` automatically (step 4 in the default order), so you typically don't need to construct it manually unless you want to enforce that environment variables are the only source.

    <Note>
      For backwards compatibility, the SDK also checks the non-standard `AWS_SECRET_KEY` variable if `AWS_SECRET_ACCESS_KEY` is not present. It logs a warning when this fallback is used.
    </Note>
  </Accordion>

  <Accordion title="InstanceProfileAWSCredentials">
    `InstanceProfileAWSCredentials` retrieves credentials from the EC2 Instance Metadata Service (IMDSv2). It automatically refreshes credentials before they expire — the SDK pre-empts expiry by 15 minutes and retries every hour.

    This credential type is intended for use when you want to explicitly target an instance profile by role name. The credential provider chain uses `DefaultInstanceProfileAWSCredentials` internally as a singleton; use `InstanceProfileAWSCredentials` when you need to target a specific role or when constructing an explicit `AmazonServiceClient`.

    ```csharp
    using Amazon.Runtime;
    using Amazon.S3;

    // Credentials retrieved from IMDSv2 for the role attached to this EC2 instance
    var credentials = new InstanceProfileAWSCredentials();
    var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USEast1);
    ```

    ```csharp
    // Target a specific IAM role by name
    var credentials = new InstanceProfileAWSCredentials("my-ec2-role");
    var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USEast1);
    ```

    <Note>
      `InstanceProfileAWSCredentials` is only meaningful when your code runs on an EC2 instance with an IAM role attached. On other environments it will fail to reach the metadata endpoint.
    </Note>
  </Accordion>

  <Accordion title="AssumeRoleAWSCredentials">
    `AssumeRoleAWSCredentials` automatically calls STS `AssumeRole` to obtain short-lived credentials for a target IAM role. The SDK refreshes them transparently before they expire, so your code never needs to manage the token lifecycle.

    Use this when:
    - Your application needs to act under a different role than its current identity.
    - You are doing cross-account access.
    - You want to apply least-privilege access per operation.

    ```csharp
    using Amazon.Runtime;
    using Amazon.S3;

    // Source credentials — the identity that is allowed to call STS AssumeRole
    AWSCredentials sourceCredentials = new EnvironmentVariablesAWSCredentials();

    var roleCredentials = new AssumeRoleAWSCredentials(
        sourceCredentials: sourceCredentials,
        roleArn: "arn:aws:iam::123456789012:role/MyTargetRole",
        roleSessionName: "my-app-session"
    );

    var client = new AmazonS3Client(roleCredentials, Amazon.RegionEndpoint.USEast1);
    ```

    You can pass an `AssumeRoleAWSCredentialsOptions` object to set additional STS parameters such as `ExternalId`, `DurationSeconds`, or an inline session policy:

    ```csharp
    using Amazon.Runtime;

    var options = new AssumeRoleAWSCredentialsOptions
    {
        ExternalId = "my-external-id",
        DurationSeconds = 3600,
        MfaSerialNumber = "arn:aws:iam::123456789012:mfa/user"
    };

    var roleCredentials = new AssumeRoleAWSCredentials(
        sourceCredentials: sourceCredentials,
        roleArn: "arn:aws:iam::123456789012:role/MyTargetRole",
        roleSessionName: "my-app-session",
        options: options
    );
    ```
  </Accordion>

  <Accordion title="AnonymousAWSCredentials">
    `AnonymousAWSCredentials` produces unsigned requests. Use this only for AWS services or endpoints that allow unsigned access, such as public S3 objects with a bucket policy that permits anonymous reads.

    ```csharp
    using Amazon.Runtime;
    using Amazon.S3;

    var credentials = new AnonymousAWSCredentials();
    var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USEast1);

    // Access a public object without signing the request
    var response = await client.GetObjectAsync("my-public-bucket", "public-object.txt");
    ```

    The credential provider chain returns `AnonymousAWSCredentials` automatically when no other credentials are found and you pass `fallbackToAnonymous: true` to `FallbackCredentialsFactory.GetCredentials()`.
  </Accordion>
</AccordionGroup>

## Loading credentials from a shared credentials file

The `SharedCredentialsFile` class reads the INI-formatted `~/.aws/credentials` and `~/.aws/config` files that the AWS CLI writes. It supports all profile types: Basic, Session, AssumeRole, SSO, and credential process.

```csharp
using Amazon.Runtime;
using Amazon.Runtime.CredentialManagement;

var credentialsFile = new SharedCredentialsFile(); // reads ~/.aws/credentials by default

CredentialProfile profile;
if (credentialsFile.TryGetProfile("dev", out profile))
{
    AWSCredentials credentials = profile.GetAWSCredentials(credentialsFile);
    var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USWest2);
}
```

To read a credentials file at a custom path, pass the file path to the constructor:

```csharp
var credentialsFile = new SharedCredentialsFile("/opt/app/aws-credentials");
```

The `SharedCredentialsFileEnvVar` constant (`AWS_SHARED_CREDENTIALS_FILE`) and `SharedConfigFileEnvVar` (`AWS_CONFIG_FILE`) environment variables can also redirect where the SDK looks for these files.

## Using CredentialProfileStoreChain

`CredentialProfileStoreChain` abstracts over both the SDK's `NetSDKCredentialsFile` (Windows only) and the cross-platform `SharedCredentialsFile`. It tries the SDK store first, then falls back to the shared credentials file.

```csharp
using Amazon.Runtime;
using Amazon.Runtime.CredentialManagement;

var chain = new CredentialProfileStoreChain();

AWSCredentials credentials;
if (chain.TryGetAWSCredentials("production", out credentials))
{
    var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USEast1);
}
else
{
    throw new InvalidOperationException("The 'production' profile was not found.");
}
```

To search only a specific credentials file and skip the SDK store, pass the file path:

```csharp
var chain = new CredentialProfileStoreChain(profilesLocation: "/etc/app/aws-credentials");

AWSCredentials credentials;
chain.TryGetAWSCredentials("staging", out credentials);
```
