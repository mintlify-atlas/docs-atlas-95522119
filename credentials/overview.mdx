title: "Credentials Overview"
description: "How the AWS SDK for .NET discovers and resolves credentials for authenticating with AWS services."
---

The AWS SDK for .NET never requires you to hard-code credentials directly into your application. Instead, it provides a credential provider chain that automatically discovers credentials from a sequence of standard locations. Understanding this chain lets you write code that works locally, in CI pipelines, and in production without changing a line.

<Warning>
  Never hard-code AWS credentials (access keys, secret keys, or session tokens) directly in your source code or commit them to version control. Hard-coded credentials create serious security risks if your code is ever shared or leaked. Use the credential provider chain or a secrets manager instead.
</Warning>

## How the credential provider chain works

When you construct an AWS service client without passing credentials explicitly, the SDK calls `DefaultAWSCredentialsIdentityResolver` (SDK v4) to walk through a prioritized list of credential sources. The first source that returns valid credentials wins; the rest are skipped.

The default search order in SDK V4 (`DefaultAWSCredentialsIdentityResolver`) is:

1. **AppConfig explicit keys / profile** *(.NET Framework only)* — keys or a profile name set in `App.config` / `Web.config`.
2. **Environment variables** — `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, and optionally `AWS_SESSION_TOKEN` and `AWS_ACCOUNT_ID`.
3. **Web identity token** — the `AWS_WEB_IDENTITY_TOKEN_FILE`, `AWS_ROLE_ARN`, and `AWS_ROLE_SESSION_NAME` environment variables (used with EKS / OIDC federation).
4. **Shared credentials file and AWS config file** — the named profile resolved from `~/.aws/credentials` and `~/.aws/config`, selected by the `AWS_PROFILE` environment variable, `AWSConfigs.AWSProfileName`, or the `default` profile as a fallback.
5. **Container / instance metadata** — ECS task credentials (`AWS_CONTAINER_CREDENTIALS_RELATIVE_URI` or `AWS_CONTAINER_CREDENTIALS_FULL_URI`), or EC2 instance profile credentials from IMDSv2 if neither container variable is set.

This order comes from `DefaultAWSCredentialsIdentityResolver` in SDK V4. Note that the legacy `FallbackCredentialsFactory` (marked `[Obsolete]` in V4) has a different order — prefer `DefaultAWSCredentialsIdentityResolver` for new code.

<Note>
  In production, prefer IAM roles attached to your compute resource (EC2 instance profile, ECS task role, Lambda execution role) over long-lived access keys. Roles provide automatically-rotating, short-lived credentials and require no secrets management on your part.
</Note>

## Setting up credentials for local development

<Steps>
  <Step title="Install the AWS CLI">
    Download and install the [AWS CLI](https://aws.amazon.com/cli/). The CLI writes credentials to the same shared file the SDK reads.

    ```bash
    aws --version
    ```
  </Step>

  <Step title="Configure a named profile">
    Run `aws configure` to create the `default` profile, or use `--profile` to create a named one:

    ```bash
    aws configure --profile dev
    ```

    This writes your access key and secret key to `~/.aws/credentials` and your region to `~/.aws/config`.
  </Step>

  <Step title="Verify the SDK can find your credentials">
    Create a minimal console app and confirm the SDK resolves credentials without error:

    ```csharp
    using Amazon.S3;

    var client = new AmazonS3Client(); // credentials resolved automatically
    var response = await client.ListBucketsAsync();
    Console.WriteLine($"Found {response.Buckets?.Count ?? 0} buckets.");
    ```
  </Step>

  <Step title="(Optional) Select a non-default profile">
    Set the `AWS_PROFILE` environment variable, or configure it in code before constructing any clients:

    ```bash
    export AWS_PROFILE=dev
    ```

    ```csharp
    AWSConfigs.AWSProfileName = "dev";
    ```
  </Step>
</Steps>

## Using FallbackCredentialsFactory directly

`FallbackCredentialsFactory` is the legacy entry point for the credential provider chain. It is marked `[Obsolete]` in SDK v4 — use `DefaultAWSCredentialsIdentityResolver` for new code — but it remains fully functional.

```csharp
using Amazon.Runtime;

#pragma warning disable CS0618 // FallbackCredentialsFactory is obsolete in v4
AWSCredentials credentials = FallbackCredentialsFactory.GetCredentials();
#pragma warning restore CS0618

var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USEast1);
```

To fall back to anonymous (unsigned) credentials when none are found instead of throwing an exception, pass `fallbackToAnonymous: true`:

```csharp
#pragma warning disable CS0618
AWSCredentials credentials = FallbackCredentialsFactory.GetCredentials(fallbackToAnonymous: true);
#pragma warning restore CS0618
```

## Passing credentials explicitly to a client

You can bypass the provider chain entirely by constructing a credentials object and passing it to the service client constructor. This is common in testing or when you need to authenticate with multiple AWS accounts simultaneously.

```csharp
using Amazon.Runtime;
using Amazon.S3;

// Static long-lived credentials (avoid in production)
var credentials = new BasicAWSCredentials("AKIAIOSFODNN7EXAMPLE", "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY");
var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USWest2);
```

```csharp
using Amazon.Runtime;
using Amazon.S3;

// Temporary credentials with a session token (e.g., from STS AssumeRole)
var credentials = new SessionAWSCredentials(
    awsAccessKeyId: "ASIAIOSFODNN7EXAMPLE",
    awsSecretAccessKey: "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY",
    token: "AQoDYXdzEJr..."
);
var client = new AmazonS3Client(credentials, Amazon.RegionEndpoint.USWest2);
```

You can also supply credentials through `ClientConfig.DefaultAWSCredentials`, which is useful when constructing a config object separately from the client:

```csharp
using Amazon.Runtime;
using Amazon.S3;

var credentials = new BasicAWSCredentials("AKIAIOSFODNN7EXAMPLE", "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY");

var config = new AmazonS3Config
{
    RegionEndpoint = Amazon.RegionEndpoint.USEast1,
    DefaultAWSCredentials = credentials
};

var client = new AmazonS3Client(config);
```
