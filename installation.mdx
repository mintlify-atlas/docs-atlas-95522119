The AWS SDK for .NET is distributed as a collection of modular NuGet packages. Each AWS service has its own package, and all service packages share a common foundation in `AWSSDK.Core`.

<Tip>
  You do not need to install `AWSSDK.Core` explicitly. Every service package declares it as a dependency, so NuGet adds it to your project automatically when you install any service package.
</Tip>

## Install a package

Choose the method that matches your workflow:

<CodeGroup>

```bash .NET CLI
dotnet add package AWSSDK.S3
dotnet add package AWSSDK.DynamoDBv2
dotnet add package AWSSDK.Lambda
```

```powershell NuGet Package Manager
Install-Package AWSSDK.S3
Install-Package AWSSDK.DynamoDBv2
Install-Package AWSSDK.Lambda
```

```xml PackageReference (.csproj)
<ItemGroup>
  <PackageReference Include="AWSSDK.S3" Version="4.*" />
  <PackageReference Include="AWSSDK.DynamoDBv2" Version="4.*" />
  <PackageReference Include="AWSSDK.Lambda" Version="4.*" />
</ItemGroup>
```

</CodeGroup>

To install through the Visual Studio NuGet Package Manager UI, right-click your project in **Solution Explorer**, select **Manage NuGet Packages**, search for the package name, and click **Install**.

## Supported target frameworks

The SDK targets multiple runtimes so you can use it across the .NET ecosystem:

| Target framework | Notes |
|---|---|
| .NET Framework 4.7.2 | Minimum supported .NET Framework version for SDK v4. |
| .NET Standard 2.0 | Compatible with Xamarin, Unity 2018.1+, and other .NET Standard 2.0 runtimes. |
| .NET 6+ | Full support including .NET 8 and later. Recommended for new projects. |

<Note>
  SDK v4 dropped support for .NET Framework versions earlier than 4.7.2 and .NET Standard versions earlier than 2.0. If you need to target those runtimes, use SDK v3.
</Note>

## ASP.NET Core integration

For ASP.NET Core applications, install the `AWSSDK.Extensions.NETCore.Setup` package to register AWS service clients in the dependency injection container:

```bash
dotnet add package AWSSDK.Extensions.NETCore.Setup
```

Then register and inject clients in `Program.cs`:

```csharp
using Amazon.Extensions.NETCore.Setup;
using Amazon.S3;

var builder = WebApplication.CreateBuilder(args);

// Register AWS options from appsettings.json or environment
builder.Services.AddDefaultAWSOptions(builder.Configuration.GetAWSOptions());

// Register individual service clients
builder.Services.AddAWSService<IAmazonS3>();

var app = builder.Build();
```

Inject `IAmazonS3` into your controllers or services using standard constructor injection:

```csharp
public class StorageService
{
    private readonly IAmazonS3 _s3;

    public StorageService(IAmazonS3 s3)
    {
        _s3 = s3;
    }

    public async Task<IEnumerable<string>> ListBucketsAsync()
    {
        var response = await _s3.ListBucketsAsync();
        return (response.Buckets ?? []).Select(b => b.BucketName);
    }
}
```

Configure the AWS region and profile in `appsettings.json`:

```json
{
  "AWS": {
    "Region": "us-east-1",
    "Profile": "default"
  }
}
```

## Modular package structure

Each service package follows a consistent naming scheme: `AWSSDK.<ServiceName>`. The package contains the service client, request and response model types, and a Roslyn code analyzer that validates property values against the service model at compile time.

```
Your project
├── AWSSDK.S3              (Amazon S3 client + models)
│   └── AWSSDK.Core        (credentials, retry, HTTP pipeline — transitive)
├── AWSSDK.DynamoDBv2      (DynamoDB client + models)
│   └── AWSSDK.Core        (shared, resolved once by NuGet)
└── AWSSDK.Extensions.NETCore.Setup  (optional DI helpers)
    └── AWSSDK.Core
```

Because all service packages declare a compatible version range of `AWSSDK.Core`, NuGet resolves a single shared copy. You only need to pin the `AWSSDK.Core` version explicitly if you require a specific bug fix or feature.

## AWS Common Runtime (CRT)

Some S3 features require the AWS Common Runtime bindings:

- S3 Multi-Region Access Points
- S3 Object Integrity checksums
- Amazon EventBridge Global Endpoints

Install the CRT integration package to enable these features:

```bash
dotnet add package AWSSDK.Extensions.CrtIntegration
```

Without this package, calling these features raises an error at runtime indicating that the AWS Signature V4a signer is missing.
